# UI/UX SaaS

## Principios
- SmartLot debe sentirse como una herramienta empresarial moderna: clara, densa y facil de escanear.
- Inspiracion visual: Linear y Vercel, con minimalismo, buen contraste, dark mode y jerarquia tipografica sobria.
- Priorizar workflows reales de gestion de estacionamientos sobre pantallas decorativas.
- Cada pantalla debe responder rapido a estas preguntas: que esta pasando, que requiere atencion y que accion sigue.

## Buenas Practicas
- Usar layouts con sidebar, topbar, filtros, tablas y paneles de resumen.
- Mantener tarjetas discretas, bordes sutiles y radios moderados.
- Usar estados vacios, loading, error y datos parciales.
- Disenar tablas con columnas alineadas, acciones consistentes, sticky headers cuando aplique y buen espaciado.
- En mobile, convertir dashboards complejos en secciones apiladas y tablas en vistas responsivas escaneables.

## Ejemplos
- Dashboard: ocupacion actual, ingresos del dia, tickets activos, alertas operativas.
- Tabla: patente, zona, hora de ingreso, duracion, estado, importe, acciones.
- Filtros: fecha, playa, estado, tipo de vehiculo.

## Patrones a Evitar
- Landing pages cuando se necesita una herramienta operativa.
- Gradientes fuertes, decoracion excesiva o paletas de un solo color.
- Cards dentro de cards.
- Texto explicando la UI en lugar de una UI clara.
- Botones sin jerarquia visual o acciones destructivas poco diferenciadas.
