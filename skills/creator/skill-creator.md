# Skill Creator

## Proposito
Crear y mantener skills cortas, accionables y reutilizables para SmartLot.

## Reglas
- Escribir instrucciones como reglas de trabajo, no como documentacion teorica.
- Mantener cada skill enfocada en un tema concreto.
- Usar ejemplos breves cuando ayuden a tomar decisiones de implementacion.
- Priorizar patrones existentes del proyecto antes de introducir nuevas convenciones.
- Actualizar una skill solo cuando la regla sea estable y aplicable a mas de una tarea.

## Buenas Practicas
- Incluir secciones como `Principios`, `Buenas Practicas`, `Patrones a Evitar` y `Ejemplos`.
- Usar bullets cortos y lenguaje directo.
- Separar reglas por frontend, backend, producto y sistema visual.
- Mantener las skills independientes para poder combinarlas segun la tarea.

## Patrones a Evitar
- Convertir una skill en un README largo.
- Guardar decisiones temporales o bugs puntuales como reglas permanentes.
- Duplicar la misma regla en muchas skills.
- Agregar archivos de skill que no tengan uso claro.
